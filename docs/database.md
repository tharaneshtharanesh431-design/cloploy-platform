# Cloploy Database Design

## Collections

### users
- identity: name, email, password hash, avatar
- access: role, email verification
- integrations: githubId, githubUsername, githubAccessToken
- recovery: resetPasswordToken, resetPasswordExpiresAt
- tenancy: organizationId
- usage: monthlyBuilds, monthlyDeployments, storageGb

### organizations
- name, slug, ownerId
- members[] with role
- billingPlan
- ssoEnabled
- secrets[]

### projects
- name, slug, description
- ownerId, organizationId
- sourceType, repositoryUrl, defaultBranch
- framework, language, runtime
- autoGeneratedDockerfile
- deploymentStrategy
- status
- latestDeploymentId
- environmentVariables[]
- aiSummary, aiRecommendations[]

### deployments
- projectId, initiatedBy
- version, status
- buildLogs[]
- qualityGate
- docker metadata
- infrastructure metadata
- monitoring links
- aiDiagnostics
- startedAt, finishedAt

### audit_logs
- actorId
- action
- entityType, entityId
- metadata
- ipAddress
